# AutoSourcingSeller 📦

**リサーチから出品準備まで、スマホで3タップ。**
物販がはじめての人でも、AIが「利益の出る商品」を探して、出品文まで全部書いてくれるアプリです。

---

## 🌸 はじめての方へ(これだけ読めばOK)

このアプリでやることは、たったの3ステップです。

### ステップ1: さがす 🔍
「リサーチ」タブで、気になる言葉を入れて「リサーチ開始」を押すだけ。

> 例: 「韓国コスメ」「北欧インテリア」「子育て便利グッズ」

AIが今の相場をネットで調べて、**利益がいくら出るか・どこで仕入れるか・リスクはどのくらいか**まで全部計算して提案してくれます。「リスク低」のバッジがついた商品から始めるのがおすすめです。

### ステップ2: えらぶ ✅
気に入った商品の「この商品で出品準備」を押すと、AIが**タイトル・説明文・価格・タグ**を自動で書いてくれます。確認画面で気になるところがあれば直して、「OK」を押すだけ。

### ステップ3: はる 📋
「タイトルをコピー」「説明文をコピー」ボタンを押して、メルカリなどの出品ページに貼るだけ。**30秒で出品完了**です。

> 📷 **写真について**: 出品には必ずご自身で撮影した写真を使ってください。商品カードの「仕入れ先ページ」リンクは購入と参考確認用です。仕入れ先の写真をそのまま転載すると規約違反になることがあります。

### ステップ4: 売れたら発送 📦
「売れた!🎉」を押すと、発送までのチェックリストが出てきます。

- **無在庫の商品**: 仕入れ先で購入 → 到着を待つ → 検品・梱包 → 発送、の4ステップ
- **在庫ありの商品**: 梱包 → 発送、の2ステップ

全部チェックすると「発送完了にする」ボタンが押せるようになります。**押し間違えても大丈夫**——各画面に「戻す」ボタンがあります。

- 「対応中」→「出品中に戻す」
- 「完了」→「対応中に戻す」

発送が終わったら、他のショップに出している同じ商品を必ず取り下げてください(完了カードにリンクが出ます)。ダッシュボードには発送完了分の利益が自動で積み上がっていきます。

---

## 💡 2つのモード

| モード | 意味 | 向いている人 |
|--------|------|------------|
| **無在庫** | 売れてから仕入れる | 在庫を抱えたくない・リスクゼロで始めたい |
| **在庫あり** | 先に仕入れて売る | 発送を早くしたい・利益を大きくしたい |

最初は**無在庫**から始めると、お金が減る心配がありません。

---

## ⚠️ 大事なお約束(アカウントを守るために)

- **無在庫の商品は、出品する前に必ず1個仕入れてください。** メルカリ・PayPayフリマは「所持していない商品」の出品を規約で禁止しています。「リサーチ」タブの提案カードにも毎回この注意が表示されます。
- **売れたら、他のショップに出している同じ商品を必ず取り下げてください。**
  (「完了」の商品カードに、確認リンクのボタンが出ます)
- **出品写真は必ずご自身で撮影したもの**を使ってください。仕入れ先の写真をそのまま転載すると規約違反になることがあります。
- このアプリは「自動出品ボット」ではありません。メルカリやPayPayフリマは自動ツールを規約で禁止していて、**使うとアカウントが凍結されます**。だからこのアプリは「AIが全部書く → あなたはコピーして貼るだけ」という一番安全な形にしてあります。

## 🛠 セットアップ(管理者向け)

### 1. Vercelにデプロイ
このリポジトリをVercelに接続すると自動でビルドされます。

- Framework Preset: **Vite**
- Build Command: `npm run build`
- Output Directory: `dist`

### 2. 環境変数の設定
Vercel → Settings → Environment Variables に以下を追加:

| Name | Value |
|------|-------|
| `VITE_ANTHROPIC_API_KEY` | AnthropicのAPIキー |

設定後、**Redeploy** してください。

### 3. ホーム画面に追加
妻さんのiPhoneでアプリのURLを開き、共有ボタン →「ホーム画面に追加」。
アプリのように使えます。

## 🧠 運用方針(コスト設計)

| 場面 | 使うモデル | 理由 |
|------|-----------|------|
| 日常のリサーチ・出品文の生成 | **Sonnet**(アプリ内で自動) | 安くて速い。毎日使っても低コスト |
| 機能追加・大規模修正・戦略の相談 | **Fable 5**(Claudeチャット) | 設計品質が必要なときだけ起動 |

アプリ内のAPI呼び出しはすべてSonnetを使うようになっています(`src/config.js` の `OPERATION_MODEL`)。

## 📷 写真加工ツール

出品準備の確認画面、および出品中の商品カードから「＋追加」で写真を登録できます。

1. 仕入れ先のページでスクリーンショットを撮る(またはダウンロード)
2. 「＋追加」でその画像を選ぶ
3. 自動で正方形にトリミング(BASE推奨サイズに近い1000px角)
4. 明るさ・コントラストを調整
5. **指でなぞると、ロゴ・価格・URLなどを白く隠せます**
6. 「この写真を使う」→「保存」でカメラロールに保存 → BASEの出品ページにアップロード

> ⚠️ **正直な注意点**: ロゴや透かしを消しても、写真そのものの著作権(仕入れ先や撮影者のもの)はなくなりません。BASEはメルカリほど画像チェックが厳しくありませんが、「絶対に安全」ではないという前提で使ってください。**商品が届いたら、できるだけ早く自分で撮影した写真に差し替える**のが一番安全です。

## 🔔 在庫確認リマインダー(現在の仕組み)

無在庫の商品は、前回確認から3日たつと「出品中」タブに通知が出ます。「見てきた」を押すと仕入れ先ページが開いて確認日が更新され、「在庫切れだった」を押すと自動で出品停止になります。

※ 今は**アプリを開いたときに教えてくれる仕組み**です。アプリを開いていない間に自動でチェックしてプッシュ通知を送る機能は、裏側で常時動くサーバーが必要なため、Phase 2以降の課題にしています。

## 🗺 ロードマップ

- ✅ **Phase 1(このバージョン)**: リサーチ → 提案(仕入れ先リンク付き) → 出品準備(確認画面+ショップ連携)→ 発送チェックリスト(取り消し対応)→ 在庫確認リマインダー + ダッシュボード + 週次レポート
- ⬜ **Phase 2**: BASE公式APIでの自動出品・自動取り下げ(BASEのみ規約上安全に自動化可能)
- ⬜ **Phase 3**: 売上目標の逆算プラン(月30万円までの道のりを週単位で提案)
- ⬜ **Phase 4**: 仕入れ価格の変動アラート・リピート商品の自動再提案
